# ASG Website Design System Brief

**Use this document as the design brief and visual foundation for all ASG Admin Hub / website UI work.** Prefer these rules over ad-hoc styling when they conflict with older component code.

## Core Design Philosophy

The ASG website design language is built around a **modern luxury utility** aesthetic. It should feel premium, minimal, structured, and highly usable. The visual system is not decorative-first. It is **clarity-first**, with refinement layered on top.

The site should communicate:

- confidence
- professionalism
- calm control
- modern real estate sophistication
- editorial cleanliness
- operational efficiency

Everything should feel intentional, high-end, and easy to scan.

The overall tone is:

- minimal
- sharp
- architectural
- slightly soft through rounded corners
- data-forward
- elegant without looking corporate or sterile

## Visual Identity Principles

### 1. Minimal, but not empty

Layouts should breathe, but never feel underdesigned. White space is used generously. Every component should have purpose.

### 2. Structured hierarchy

Users should immediately understand:

- what the page is
- what section matters most
- what action to take
- what data is secondary

Hierarchy must come from:

- size
- weight
- spacing
- grouping
- contrast

Not from loud color or visual gimmicks.

### 3. Premium utility

The interface should feel like a luxury brand crossed with a dashboard. It is not a playful SaaS app. It is also not a generic brokerage site. It needs operational clarity with polished presentation.

### 4. Consistency over novelty

New sections should inherit the same design logic:

- same corner radii
- same spacing rhythm
- same card logic
- same typography behavior
- same button language
- same hover behavior

## Overall Style Direction

The recurring style language across the project is:

- white or warm-white surfaces
- black or near-black typography
- muted grays for secondary information
- occasional restrained accent colors
- soft shadows only when needed
- large rounded corners
- bento grid layouts
- pill-shaped interactive elements
- strong typography
- clean separation of primary vs secondary info

The site should feel like a blend of:

- high-end editorial design
- modern startup dashboard UI
- luxury property marketing
- clean Apple-like restraint
- contemporary creative portfolio polish

## Layout System

### Bento Box Philosophy

A major recurring pattern is the **bento box layout**.

Bento boxes should:

- organize information into clean modular units
- create visual rhythm
- make complex pages easier to scan
- feel premium and contemporary
- help mix stats, actions, lists, timelines, and summaries in one visual language

Bento boxes should not feel crowded. Each box must have:

- a single clear purpose
- one primary focal point
- limited text
- strong spacing
- simple internal alignment

### Bento Box Rules

- large border radius
- white or off-white background
- subtle border or shadow
- generous padding
- clear heading
- optional small supporting label
- restrained hover effects
- no heavy gradients unless explicitly needed
- no glassmorphism unless extremely subtle and context-specific

### Bento Box Content Hierarchy

Within each card:

1. label or eyebrow
2. main title/value
3. supporting data or action
4. optional footer/meta detail

## Spacing System

Spacing is critical to the ASG design language. Dense layouts weaken the brand.

Use:

- large section padding
- comfortable card padding
- strong gaps between content groups
- consistent vertical rhythm

### General spacing behavior

- tight spacing only inside small metadata clusters
- medium spacing for headings to body
- generous spacing between modules
- extra generous spacing in hero/dashboard areas

The site should never feel cramped.

## Corner Radius System

Rounded corners are a defining characteristic.

Typical corner approach:

- large rounded containers
- pill-shaped buttons and badges
- softer cards
- cleaner modern feel

### Radius guidance

- major containers: 24px to 32px
- standard cards: 20px to 30px
- buttons/badges: full pill or high radius
- inputs/search bars: pill or very rounded rectangle

Important nuance:

- some media containers should have only top corners rounded for cleaner card integration
- avoid over-rounding nested elements if it creates visual softness overload

## Typography System

Typography should feel modern, clean, bold, and premium.

### Preferred fonts used across the project

- **Outfit** (strongest overall fit)
- Satoshi
- Poppins

Outfit works well for:

- bold headers
- UI labels
- cards
- dashboards
- modern editorial feel

### Typography behavior

- bold, confident headings
- clean medium-weight subheads
- subdued body copy
- strong number styling for stats
- uppercase or small eyebrow labels where useful
- avoid overly thin weights

### Hierarchy

- page titles should feel strong and editorial
- section headings should be bold and crisp
- labels should be understated
- metadata should be smaller and grayer
- stats should be large and visually stable

### Tone of typography

The typography should make the interface feel:

- controlled
- premium
- intelligent
- modern
- easy to trust

## Color System

The project relies on a restrained monochrome-first palette.

### Primary palette

- white
- black / near-black
- soft grays
- light neutral backgrounds

### Accent usage

Color should be used sparingly and purposefully:

- live status
- active states
- alerts
- emphasis dots
- selected tabs
- small category pills

Avoid large blocks of saturated color.

### Design intent for color

Color should guide, not dominate.

Examples:

- green for live/current/progress states
- muted red dot for “Closed”
- subtle cream or off-white only when warmth is needed
- soft muted fills for small category identifiers

The default experience should still look complete in near-monochrome.

## Surface Styling

### Cards and containers

- mostly white backgrounds
- subtle border or low-contrast outline
- soft shadow only when necessary
- no harsh gray container backgrounds unless used as a page contrast field

### Shadows

Use very restrained shadows. The site should not feel like a generic template with floating cards everywhere.

Preferred shadow behavior:

- faint
- soft
- diffused
- nearly imperceptible

### Borders

Thin borders are often better than heavier shadows. Use subtle borders to define surfaces cleanly.

## Buttons and Interactive Elements

Buttons across the ASG project should feel clean, strong, and consistent.

### Button style

- pill-shaped or heavily rounded
- balanced padding
- simple label structure
- no unnecessary icons unless useful
- monochrome-first
- high clarity

### Button hierarchy

**Primary buttons:**

- dark fill with light text
- used sparingly for main actions

**Secondary buttons:**

- light fill or bordered
- used for navigation and supporting actions

**Utility/action pills:**

- compact
- clean
- evenly sized where grouped

### Button behavior

Hover states should be subtle:

- slight background shift
- slight border emphasis
- maybe slight lift
- no aggressive animations

## Navigation Style

Navigation should feel simple, elegant, and modern.

### Preferred navigation patterns

- pill-shaped nav bars
- segmented navigation
- clean tab systems
- restrained sticky header behavior
- strong active state clarity

### Navigation should be

- obvious
- minimal
- visually aligned with the rest of the UI
- easy to scan

Avoid:

- cluttered menus
- tiny text links
- overcomplicated dropdown structures

## Search and Input Styling

Search is a recurring pattern in the project, especially in admin and asset hub experiences.

### Search inputs should feel

- premium
- clean
- roomy
- rounded
- intuitive

### Search styling

- pill or soft rounded rectangle
- subtle border
- clean placeholder text
- no heavy background tint
- optional icon aligned left
- enough height to feel polished

Inputs should visually belong to the rest of the design system, not default browser styling.

## Status, Tags, and Pills

Small pills and badges are heavily used.

### Badge style

- pill-shaped
- compact
- clean text
- subtle fill or border
- used to label status, category, or context

### Example behavior

- white badge on image/cards for listing status
- red dot next to “Closed”
- muted tone for secondary categories
- green or highlighted style for current timeline steps

These should feel refined, not loud.

## Imagery and Media Treatment

Photography is central to the ASG brand. Image presentation needs to feel polished.

### Image rules

- large, clean, immersive
- rounded thoughtfully
- never cramped
- integrated into card/container structure
- high visual quality
- no awkward cropping

### Preferred media treatment

- cover images with top-only rounded corners when inside cards
- content image aligned flush with card edge when appropriate
- avoid excessive image overlays
- keep captions minimal

Matterport, floor plans, videos, and asset links should visually match the card language of the rest of the site.

## Data and Dashboard Styling

Admin/dashboard experiences are a major part of this project. They must still feel on-brand.

### Dashboard design rules

- dashboard should look like part of the website, not separate software
- bento layout is preferred
- stats should be large and clear
- supporting info should be simplified
- avoid dense tables unless necessary
- prioritize summaries, snapshots, and quick actions

### Stats cards should emphasize

- one main metric
- one supporting context line
- one optional comparison or breakdown

### Dashboard should feel

- executive
- clean
- fast to read
- designed for real use, not just display

## UX Behavior Principles

### 1. Immediate clarity

Every section should explain itself visually in seconds.

### 2. Reduced friction

The user should always know:

- where they are
- what they can do
- what matters most

### 3. Scan-first design

People should be able to skim pages and still understand the structure.

### 4. Familiar component logic

Repeated patterns matter:

- cards should behave similarly
- tabs should look consistent
- button systems should be stable
- headings should follow a rhythm

### 5. Beautiful, but functional

Do not sacrifice usability for effect.

## Motion and Interaction

Motion should be minimal and elegant.

Use:

- subtle hover lift
- faint fade transitions
- soft color transitions
- understated active-state animation

Avoid:

- springy excessive motion
- flashy transitions
- gimmicky interactions
- unnecessary parallax
- animation that slows usability

## CSS Personality

When generating CSS for this project, the code should aim for:

- elegance
- maintainability
- consistency
- restrained polish

The CSS should favor:

- consistent variables for spacing/radius/color
- reusable utility classes where helpful
- clean grouping of component styles
- careful mobile responsiveness
- no visual noise

## Responsive Design Intent

On mobile, the design should remain premium and breathable.

### Mobile behavior

- bento grids stack cleanly
- spacing remains generous
- text remains readable
- tap targets stay large
- buttons stay pill-shaped and usable
- hierarchy stays intact

Nothing should collapse into clutter.

## Cursor Instruction Layer

Use when generating design-focused code:

### Design rules for Cursor

Build all UI in a clean luxury-minimal style consistent with the ASG website system. Use large rounded bento cards, monochrome-first color palette, bold modern typography using **Outfit** where possible, pill-shaped buttons and navigation, subtle borders instead of heavy shadows, clean spacing, premium dashboard-like hierarchy, restrained accent colors for status and emphasis only, and highly polished responsive layouts. Prioritize scanability, elegance, white space, and visual consistency across all components. Avoid generic SaaS styling, heavy gradients, glassmorphism, bright colors, cramped layouts, and default-looking form elements.

### Compressed prompt (short version)

Design this in the ASG website style: modern luxury minimal, black/white with restrained accent color, large rounded bento cards, Outfit typography, bold clean headers, pill-shaped buttons and tabs, subtle borders, soft shadows only if needed, generous spacing, editorial hierarchy, premium dashboard feel, highly polished responsive layout, and extremely clean UX. Keep everything refined, monochrome-first, and consistent with a high-end real estate brand plus modern ops dashboard aesthetic.

## What To Avoid

Do not generate:

- generic Bootstrap-looking layouts
- overly corporate SaaS UI
- bright blue primary-button-heavy interfaces
- tight spacing
- tiny fonts
- overdesigned gradients
- excessive shadows
- glassmorphism-heavy cards
- too many icon decorations
- cluttered headers
- inconsistent radii
- random accent colors
- noisy backgrounds
- dense tables as default UI
- mismatched button styles
- default form styling

## Final Design Standard

Every page or component should pass this test:

It should look like it belongs inside one unified ASG digital ecosystem:

- premium
- architectural
- minimal
- modern
- highly usable
- operationally smart
- visually calm
- confidently designed
